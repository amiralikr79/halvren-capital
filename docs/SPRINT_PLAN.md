# Halvren — 7-Sprint Plan

48-hour autonomous build window. Seven sprints. Each ships something the principal can see in production before the next begins. North star: **machine breadth, human conviction.**

Status legend: ☐ pending · ◐ in-flight · ☑ shipped

---

## Sprint 1 — Brand + Ops · ☑ shipped

Locked the foundation so the rest of the work can move without re-litigating taste or process.

- ☑ `/docs/HALVREN_BRAND.md` — voice, forbidden phrases, color tokens, typography, motion, north star.
- ☑ `/docs/OPERATING_RULES.md` — decide-log-ship; no TODOs; green-build-before-commit; `halvren:` commit prefix; push-to-main-on-green.
- ☑ `/docs/DECISIONS.md` — append-only log seeded with Sprint 1 calls.
- ☑ `/docs/SPRINT_PLAN.md` — this file.
- ☑ Commit + push: `halvren: lock brand constitution + sprint plan`.

**Definition of done:** the four docs exist on `main`, the commit lands, and any future copy/code decision can be measured against the constitution.

## Sprint 3 — 10 authority notes · ☐ pending

Ten short authority notes (~700–1,200 words each). Not full research pieces. Single-claim essays that demonstrate the desk's reading. Each one ends with a line that lands.

- ☐ Note 1 — _What heavy-oil cost curves actually tell you_ (Canadian energy).
- ☐ Note 2 — _Uranium's second-mover problem_ (Cameco / Kazatomprom framing).
- ☐ Note 3 — _The dividend that isn't_ (pipelines, payout ratio vs. rate-base growth).
- ☐ Note 4 — _Buybacks at the trough, issuance at the top_ (the simplest tell).
- ☐ Note 5 — _Reading insider purchases the boring way_ (open-market vs. options).
- ☐ Note 6 — _Why "first-quartile" is a sales word, not an arithmetic one_.
- ☐ Note 7 — _The fertilizer cycle in three numbers_ (potash, nitrogen, phosphate).
- ☐ Note 8 — _Forest products, the cycle nobody writes about_.
- ☐ Note 9 — _Midstream as the regulated cash machine_ (Enbridge / TC).
- ☐ Note 10 — _What a 2015 or a 2020 balance sheet tells you about a 2026 management team_.

Each note: dedicated HTML page under `/notes/<slug>.html`, OG image generated by extending `scripts/og_research.py`, indexed in a new `/notes` listing page.

**Definition of done:** ten notes live, each on its own URL, OG images generated, sitemap updated, `llms.txt` and `llms-full.txt` regenerated to include them.

## Sprint 4 — Design system + Constellation hero · ☐ pending

Codify the design system in CSS and replace the homepage hero with the Constellation.

- ☐ Promote color tokens from `--color-*` to the canonical aliases (`--bg`, `--ink`, `--muted`, `--line`, `--green`, `--red`, `--gold`) with the existing names kept as synonyms.
- ☐ Install Cormorant Garamond (primary) with Lora (fallback) from Google Fonts. Apply to `h1, h2`. Body stays.
- ☐ Extract repeated patterns into a small set of utility classes (`.eyebrow`, `.lede`, `.land`, `.rule-gold`). One file, one CSS, no framework.
- ☐ Build the **Constellation hero**: a quiet, minimal node-graph of the 20 covered operators, edges by sector, mounted above the fold. Static SVG generated at build time from `coverage.json`. No JS animation beyond the existing reveal.
- ☐ Replace the current hero block with the constellation while keeping the eyebrow + sub-line + meta.

**Definition of done:** light and dark mode both pass a visual review against the brand doc; Cormorant loads with `font-display: swap` and a system-serif fallback; the constellation renders with JS disabled.

## Sprint 5 — Checklist Live tool · ☐ pending

Turn the static homepage checklist into a working tool. Reader pastes a ticker; the desk's checklist runs against the most recent filings and returns a deterministic, citation-backed scorecard.

- ☐ Extend `api/checklist/score.js` to accept arbitrary tickers (not just the coverage list), with a strict rate limit via Upstash.
- ☐ Add a public UI at `/checklist/live` — paste a ticker, see the ten questions, see each as `pass / fail / unclear`, with a single citation per answer.
- ☐ Cache results for 24h per ticker in Upstash Redis to keep costs deterministic.
- ☐ OG image per scored ticker (`/checklist/score/<ticker>` already exists as a rewrite — extend it).
- ☐ Honest failure modes: if the model can't find a filing, the answer is _unclear_, not _pass_. The UI shows it as such, with the reason.

**Definition of done:** a stranger can paste a Canadian or U.S. operator ticker and get a complete, cited scorecard within ~15s, with caching for repeat calls.

Rewrite the About page against the brand doc. Run a copy pass across every page. Add a quiet Digest ticker to the homepage.

- ☐ `/about` — rewrite top-to-bottom. One desk, one principal. No bio bullet list — prose only. Cut the Substack plug down to one line.

- ☐ Add the **Digest ticker** to the homepage — a single-line, monospace strip below the hero showing the last digest's filing count, model flags, and any names promoted to principal review. Pulled from `digest/latest.json`. Updates on each digest build.
- ☐ Run the grep guard from `OPERATING_RULES.md` on the whole site. Zero hits.

**Definition of done:** zero forbidden phrases in the live HTML; the About page is one page of prose; the Digest ticker reads correctly with and without JS.

## Sprint 7 — SEO / AEO + QA · ☐ pending

The last sprint is hygiene. SEO for search; AEO for the model-driven discovery layer; QA across the surface.

- ☐ JSON-LD: extend `Organization` and `Person` schema on every page that needs it. Add `Article` schema to every note and research piece. Add `BreadcrumbList` to deep pages.
- ☐ `llms.txt` + `llms-full.txt` regeneration via `scripts/build_machine_readable.py`. Coverage list, all notes, all letters, the brand north-star line.
- ☐ Sitemap regenerate. OG images audit (every page has one, every one is ≤ 200KB, dimensions correct).
- ☐ Lighthouse pass: 100 / 100 / 100 / 100 on the homepage in light and dark mode. Anything under 100 is a bug.
- ☐ Broken-link sweep — internal and external.
- ☐ Reduced-motion sweep — every reveal respects `prefers-reduced-motion`.
- ☐ Final read-through against the brand doc. Anything that doesn't pass, fix or remove.

**Definition of done:** Lighthouse green, zero broken links, sitemap + llms files current, JSON-LD validates on every page. The site is shippable to a Buffett shareholder letter audience without an apology.

---

## Sequencing notes

- Sprints 2 and 3 can interleave: a new coverage entry can ship the same day as the authority note that motivates it.
- Sprint 4 (design system) is a hard cut. Land it before Sprint 6's copy pass so the rewrite happens against the new tokens.
- Sprint 5 (Checklist Live) is the only sprint with meaningful runtime risk. Budget Upstash limits up front; log the call in `DECISIONS.md`.
- Sprint 7 is non-negotiable. Hygiene at the end is hygiene that survives.
