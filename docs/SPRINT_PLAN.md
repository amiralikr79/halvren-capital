## Sprint 5 — Checklist Live tool · ☑ shipped

Built the streaming Halvren Checklist engine, the standalone interactive page, and the homepage embed. Architecture: SSE JSON-Lines from a Vercel function backed by Sonnet 4.6, Upstash Redis for cache + rate-limit, Yahoo Finance public endpoints for fundamentals, four hand-written anchor examples for voice calibration.

Frontend:
- ☑ `/checklist/live` standalone page — italic hero line, single input, 10 rows that fade in as the stream arrives, scorecard that slides in on complete, trust footer.
- ☑ Homepage embed below the static Checklist section — same form, same flow, results render inline.
- ☑ Shared stylesheet at `/checklist/live/live.css` with token-alias fallbacks so the embed works on the homepage (which does not load `page.css`).
- ☑ Deep links: `/checklist/live/<TICKER>` rewrites to the page, the client reads the URL, auto-runs the same flow, and replaces the URL after fresh runs.
- ☑ Shareable URL with a copy-to-clipboard button on every completed run.

Backend:
- ☑ `api/checklist/[ticker].js` — SSE streaming endpoint. Emits `meta`, then `line` (×11), then `complete`, then `end`. Errors emit `error` then `end`.
- ☑ `api/checklist/_lib.js` — shared helpers: ticker normalisation, Redis singleton + 10/IP/hour sliding-window rate limit, 24-hour cache, Yahoo Finance fundamentals fetch with `.TO`/`.V` retry, system prompt assembly with the four anchor examples.
- ☑ `data/checklist-examples.json` — four hand-written voice anchors (CCO, CNQ, AG, ENB). Each is the literal output the engine is calibrated to produce.
- ☑ `scripts/prewarm_checklist_cache.py` — sequential warmer that hits the live endpoint for all 21 published operators with an 8-second gap, ready to run post-deploy with `HALVREN_BASE_URL`.
- ☑ vercel.json: rewrite `/checklist/live/:ticker` to the standalone HTML; function config for `[ticker].js` with `maxDuration: 60` and the required `includeFiles`.

Guarantees on every result:
- ☑ Trust footer: "Generated by the Halvren Checklist engine. Not yet reviewed by the principal. The full review takes longer than a page load."
- ☑ Never a price target, buy/sell, or short-term forecast (system-prompt rule).
- ☑ Graceful sparse path: if fundamentals are unavailable, the engine emits the canonical "would need to read the primary filings" template and caches it like any other result.
- ☑ Rate-limit copy: "The desk reads slowly. Come back in an hour, or read what we've already published →" linking to /research.

Manual test pass: ticker regex accepts CCO, NVDA, SHOP, ATD, CIX.TO, ATD.TO. The four-example JSON parses cleanly and every example produces 10 answer lines + a scorecard line through the line-parser logic that the API uses.

**Definition of done:** ✅ a stranger can paste any Canadian or U.S. operator ticker and watch the 10 questions stream in; ✅ each row carries a verdict dot in --green / --gold / --red; ✅ scorecard slides in on complete with chips, overall sentence, and Substack CTA; ✅ shareable `/checklist/live/<TICKER>` URL; ✅ 10-per-IP-per-hour rate limit; ✅ 24-hour Upstash cache; ✅ pre-warm script for the published coverage cohort; ✅ trust footer on every result.
